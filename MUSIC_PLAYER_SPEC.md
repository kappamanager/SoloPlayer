# MekaPlayer — PWA音楽プレイヤー仕様書

## 概要
スマホ内のmp3/wavファイルを再生するPWA（Progressive Web App）。広告なし。Android Chrome上で動作し、ホーム画面に追加してアプリとして使用する。PC Chromeでも動作確認可能。

## 技術構成
- HTML + CSS + JavaScript（単一ファイル or 最小構成）
- PWA（Service Worker + Web App Manifest）
- Media Session API（バックグラウンド再生 + 通知バー操作）
- IndexedDB（プレイリスト情報・設定の永続保存。曲ファイル本体は保存しない）
- File System Access API or input要素（フォルダ読み込み）

## フォルダ構造（ユーザー側）
```
読み込み親フォルダ/
├── TOEIC/
│   ├── cover.jpg
│   ├── [003]_730点以上攻略問題_Part1.mp3
│   └── ...
├── J-POP/
│   ├── cover.jpg
│   ├── 3月9日.mp3
│   └── ...
├── BGM/
│   ├── cover.jpg
│   ├── 30. Ikson - October Sun.mp3
│   └── ...
└── ...
```
- 2階層固定：親フォルダ > 区分フォルダ > 曲ファイル + cover.jpg
- 親フォルダ直下の曲ファイルは無視する
- 対応形式：mp3, wav

## アルバムアート
優先順位：
1. 曲ファイルのメタデータ（ID3タグ）に埋め込み画像がある → それを使う
2. 同じ区分フォルダ内に `cover.jpg` がある → そのフォルダの曲全部に適用
3. どちらもない → デフォルトアイコン表示

- 対応画像形式：jpgのみ
- ファイル名：`cover.jpg` 固定（cover.png等は対象外）

## 画面構成

### タブ
2つのみ：
- **曲**：全曲一覧を表示
- **プレイリスト**：プレイリスト一覧を表示

### 曲タブ
- 全曲をファイル名順（昇順）で一覧表示
- 各曲の表示項目：アルバムアート（サムネイル）、曲名（ファイル名）、再生時間
- 曲タップで再生開始
- 検索機能あり（曲名でフィルタ）

### プレイリストタブ
- **自動プレイリスト**：区分フォルダ名がそのままプレイリスト名になる。フォルダ読み込み時に自動生成。編集不可（フォルダの中身がそのまま反映）。サムネイルにそのフォルダのcover.jpgを表示
- **手動プレイリスト**：ユーザーが任意に作成。複数フォルダの曲を横断して追加可能。名前は自由。曲の追加・削除・並び替えができる。サムネイルはプレイリスト内の最初の曲のアルバムアートを使用
- 手動プレイリストはIndexedDBに保存。フォルダ再読み込みしても消えない
- 各プレイリストカードの表示項目：サムネイル画像、プレイリスト名、曲数

### ミニプレイヤー（画面下部固定）
- 現在再生中の曲名
- アーティスト名（メタデータにあれば。なければ非表示）
- 再生/一時停止ボタン
- 前の曲 / 次の曲ボタン
- シャッフル状態表示

## 再生機能

### 基本操作
- 再生 / 一時停止
- 前の曲 / 次の曲
- シークバー（再生位置の変更）

### シャッフル
- 再生中のコンテキスト（曲タブ全体 or プレイリスト内）に対して動作
- 曲タブで再生中 → 全曲シャッフル
- プレイリスト内で再生中 → そのプレイリスト内だけシャッフル
- ON/OFFトグル

### リピート
- 3段階切り替え：リピートOFF → 全曲リピート → 1曲リピート
- シャッフルと同様、再生中のコンテキストに対して動作
- 曲タブで再生中 → 全曲ループ
- プレイリスト内で再生中 → プレイリスト内ループ

### 再生順
- デフォルトはファイル名順（昇順）

### バックグラウンド再生
- Media Session APIを使用
- Android通知バーから再生/一時停止/前/次の操作が可能
- アルバムアートがあれば通知バーにも表示

## データ永続化（IndexedDB）

### 保存するもの
- 手動プレイリスト（名前、含まれる曲のリスト）
- 設定（シャッフルON/OFF、リピートモード、最後に再生した曲の情報）
- ※将来的に拡張可能な構造にしておく

### 保存しないもの
- 曲ファイル本体（端末内に存在するため）
- 自動プレイリスト（フォルダ読み込み時に毎回生成）

## フォルダ読み込みフロー
1. アプリ起動
2. 「フォルダを選択」ボタンを表示
3. ユーザーが親フォルダを選択
4. 親フォルダ内の区分フォルダを走査
5. 各区分フォルダ内のmp3/wavファイルとcover.jpgを読み込み
6. 自動プレイリストを生成
7. 曲タブに全曲表示
8. ※ブラウザを閉じるかタスクキルしない限り再選択不要（開きっぱなし運用）

## デザイン
- アプリ名：**SoloPlayer**
- テーマ：アンチYouTube（赤/白の逆）。黒ベース＋高級感のある青グラデーション
- 背景：漆黒（#08080e〜#0a0a10）
- アクセント：深い青のグラデーション（#1a237e → #4a6cf7 → #7b9fff）
- テキスト：明るい青白（#c8d0f0 メイン、#7b9fff アクセント、#3d4f80 サブ）
- プログレスバー：青→水色のグラデーション
- 再生中の曲：ほんのり青く光るハイライト
- プレイヤーバー：暗闘から浮かび上がるグラデーション背景
- 全体的にプレミアム感・意識高い系のトーン
- 参考モックアップ：soloplayer_mockups.html

## 開発手順（Claude Code向け）
1. まずPCのChromeで動作するHTML/JS/CSSを作成
2. 基本再生機能 → プレイリスト → シャッフル/リピート → バックグラウンド再生の順で実装
3. PCで動作確認後、Androidで検証
4. デザイン調整は機能が安定した後に行う

## 制約事項
- PWAのためブラウザのセキュリティ制約あり。ファイルパスの永続保存はできない
- フォルダ読み込みは毎回手動（ただし開きっぱなし運用で問題なし）
- iOSは対象外（Androidのみ）
